## チェーン間の自然なブリッジ

サイコロを転がして、時々幸運にも複数のゲームで同時に勝つことを想像してください。それが本質的にコインシデントブロックです - 複数のブロックチェーン要件を同時に満たすマイニング解です。

**何が特別か：**
- **追加作業不要**：通常のマイニングを通じて自然に発生
- **数学的に保証**：バリデーターや委員会に依存しない
- **完璧な同期**：純粋な確率を通じてチェーンが接続されたまま

**クロスチェーントランザクションをいかに可能にするか：**
コインシデントブロックが発生すると、チェーン間に破られない数学的リンクが作成されます。このリンクはトラストレスブリッジとして機能し、仲介者なしでチェーン間でアセットとデータが移動できます。

## 運の数学

**簡単な類推：**
- ゾーン困難度：6を出す必要
- リージョン困難度：2つの6を出す必要
- プライム困難度：3つの6を出す必要

1つしか必要ないときに3つの6を出すことがあります - それがコインシデントブロックです！

**確率表：**

| ブロックの「運の良さ」        | 発生確率  |
| --------------------------- | -------- |
| やや運が良い（余分なゼロ1つ）  | 50%      |
| かなり運が良い（余分なゼロ2つ） | 25%      |
| とても運が良い（余分なゼロ3つ） | 12.5%    |
| 極めて運が良い（余分なゼロ4つ） | 6.25%    |

**これが重要な理由：**
委員会、バリデーター、または複雑なプロトコルに依存する他のマルチチェーンシステムとは異なり、Quaiのクロスチェーンブリッジは純粋な数学によって作成されます。信頼は不要 - 確率だけです。

## コインシデントブロックの動作方法

**重要な原則：**

1. **階層が重要**：プライムブロックは常にコインシデント（すべてのチェーンを満たす）
2. **独立性の保持**：ゾーンは待機することなくブロックを生成可能
3. **アトミック有効性**：ブロックはすべての場所で有効でなければ、どこでも無効
4. **完璧な同期**：すべてのチェーンがコインシデントブロックを同時に追加

**チェーン階層：**
- **プライムブロック**：常にコインシデント（すべての錠を開くマスターキーのような）
- **リージョンブロック**：それらのゾーンとコインシデント
- **ゾーンブロック**：通常は独立、時々コインシデント

**なぜ異なるブロック時間？**
- プライム：~20秒（遅いが超安全）
- リージョン：~10秒（バランス型）
- ゾーン：~5秒（ユーザーにとって高速）

この設計により、ゾーンは高速で動作しながら、コインシデントブロックを通じて定期的同期を維持することが保証されます。

## ノードの実行

**最小要件 - 「スライス」：**
マイナーが実行できる最小ノードはスライスノードです。トラストレスに参加するために必要なもの：
- プライムチェーン（グローバル協調）
- 1つのリージョンチェーン（地域協調）
- 1つのゾーンチェーン（トランザクションが発生する場所）

この「スライス」により、他の誰も信頼することなくトランザクションを検証するために必要なすべてが提供されます。

**システムの美しさ：**
コインシデントブロックは、複数のブロックチェーンが同期を保ち、セキュリティを共有するためのトラストレス、自動、数学的に保証された方法を作成します - すべてマイニングの自然なランダム性を通じて。
