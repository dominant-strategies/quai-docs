## 比例分割による最適報酬配分

**2025年3月13日公開**

ビットコインの最も有名な攻撃であるセルフィッシュマイニングの発表に続いて、さまざまな研究がブロックチェーンシステムのゲーム理論的耐性を強化するメカニズムを導入しています。FruitChains などの一部の報酬メカニズムは、理論的に均衡であることが示されています。ただし、それらの保証は非現実的なパラメータを仮定しており、実際の展開設定では性能が大幅に低下します。この研究では、比例分割（PRS）と呼ばれる報酬配分メカニズムを導入し、既存の最先端技術を上回る性能を示します。十分に大きなパラメータに対して、PRS は均衡であり、最先端技術と同じ理論的保証を提供することを示します。さらに、実用的で現実的に小さなパラメータに対して、PRS は一連のメトリクスにわたって既存のすべての報酬メカニズムを上回ります。PRS を PoEM の変種の上に実装します。PoEM は、例えばビットコインと比較して、各当事者のマイニングパワーのより正確な推定を可能にするプルーフ・オブ・ワーク（PoW）プロトコルです。次に、PRS を理論的および実践的に評価します。理論面では、PRS と組み合わせた私たちのプロトコルが均衡であり、FruitChains と同様に公平性を保証することを示します。実際には、PRS を一連の既存の報酬メカニズムと比較し、マイニングパワー分布の正確な推定を仮定して、さまざまな確立されたメトリクスでそれらを上回ることを示します。最後に、「ワークシェア」と呼ばれる低ワークオブジェクトを介してパワー分布を近似することでこの仮定を実現し、近似の精度とストレージオーバーヘッドとの間のトレードオフを定量化します。

[arxiv.org](https://arxiv.org/abs/2503.10185) で完全な論文をお読みください。

## より良いプルーフ・オブ・ワーク フォーク選択ルール

**2024年2月9日公開**

プルーフ・オブ・ワークブロックチェーンのフォーク選択ルールの修正を提案します。最重チェーンを選択する代わりに、最も内在的な作業量を持つチェーンを選択します。ブロックの内在的な作業量は、おおよそそのハッシュの前面にあるゼロの数です。この修正により、必要な確認を安全に減らすことができ、確認遅延が 28.5% 改善されるか、あるいは双対的に、ブロック生産率を安全に増加させ、バニラビットコインプルーフ・オブ・ワークフォーク選択ルールと比較してスループットが 16.3% 改善されます。私たちの修正はプルーフ・オブ・ワーク不等式のレベルであるため、文献で提案されている遅延やスループットを改善する他の方法と組み合わせることができます。すでに野外に展開されているテストネットを持つシステムの本格的な実装で測定することにより、実験結果を報告します。最後に、ビットコインバックボーンモデルで新しいプロトコルのセキュリティを正式に証明します。

[iacr.org](https://eprint.iacr.org/2024/200) で完全な論文をお読みください。

## PoEM：プルーフ・オブ・エントロピー・ミニマ

**2023年3月8日公開**

ナカモトコンセンサスは堅牢なブロックチェーンシステムを可能にする上で非常に影響力があり、その構成要素の1つはいわゆる最重チェーンルール（HCR）です。このルール内では、チェーンティップの重みの計算は、難易度閾値を前の総難易度に追加することによって実行されます。現在の難易度ベースの重み付けシステムは、内在的なブロック重みを考慮していません。この論文では、エントロピー差に基づく新しいメカニズム、プルーフ・オブ・エントロピー・ミニマ（POEM）を提案します。これは、ブロックチェーンの孤立率を大幅に削減すると同時にファイナライゼーションを加速する方法で内在的なブロック重みを組み込みます。最後に、POEM はブロックチェーンをコミットされたイベントの静的で時間に依存しないシーケンスとして理解するのに役立ちます。

[arxiv.org](https://arxiv.org/abs/2303.04305) で完全な論文をお読みください。

## 階層最長チェーンルールによるスケーラブルなマルチチェーン調整

**2022年IEEE国際ブロックチェーン会議（Blockchain）で公開**

この論文では、BlockReduce を紹介します。これは、マージマイニングされたブロックチェーンの階層を通じて高い取引スループットを実現するプルーフ・オブ・ワークベースのブロックチェーンシステムで、各ブロックチェーンは全体のアプリケーション状態のパーティション上で並行して動作します。最も注目すべきは、ネットワーク内で利用可能な完全な PoW が BlockReduce のすべてのブロックチェーンに適用され、クロスブロックチェーン状態遷移がコアプロトコル内でシームレスに有効になることです。この論文では、ブロックチェーンの階層とその関連するセキュリティモデルが与えられた場合、BlockReduce がプロトコルによって運用されるブロックチェーンの数に対して取引スループットで超線形にスケールすることを示しています。BlockReduce は、各ブロックチェーンの正規チェーン（つまり、適用される状態更新を参照するブロックのチェーン）を選択するために新しいコンセンサスルールを利用します。ビットコインやその他のより伝統的なシステムでは、最長チェーンルール（LCR）は、正規チェーンが最も多くの作業を持つ有効なブロックのシーケンス（しばしば最長チェーン、またはより正確には最重チェーンと呼ばれる）であることを規定しています。BlockReduce は同様のルールに従いますが、階層内の同時ブロックの存在も考慮する必要があります。

[ieee.org](https://ieeexplore.ieee.org/document/9881846) で完全な論文をお読みください。

## Hashcore：汎用プロセッサのためのプルーフ・オブ・ワーク関数

**2019年IEEE第39回分散コンピューティングシステム国際会議（ICDCS）で公開**

過去5年間で、プルーフ・オブ・ワークブロックチェーンのマイニングに関連する報酬は大幅に増加しました。その結果、マイナーは既存の汎用ハードウェアよりもはるかに効率的にハッシュを計算できる特定用途向け集積回路（ASIC）を設計・利用することに強いインセンティブを受けています。現在、価格と入手可能性の制約により、ほとんどのユーザーが ASIC を購入・運用することは困難であり、人気のある暗号通貨の多くで総ユーザーベースに対してマイナーの数が比較的少なくなっています。この研究では、既存の汎用プロセッサ（x86 IC などの GPP）がすでに最適化された ASIC であるプルーフ・オブ・ワーク関数を構築することで、ASIC 開発の問題を逆転させることを目指しています。そうすることで、潜在的なマイナーが参加したいプルーフ・オブ・ワークシステムの ASIC をすでに所有しているか、比較的簡単に競争力のある価格で入手できることを保証します。これを達成するために、HashCore を提示します。これは、実行時に疑似ランダムに生成される「ウィジェット」で構成されるプルーフ・オブ・ワーク関数で、各ウィジェットは GPP の計算リソースにストレスをかけるように設計された汎用プロセッサ命令のシーケンスを実行します。ウィジェットは、GPP が最適化されているワークロード、例えば x86 IC の SPEC CPU 2017 ベンチマークスイートをモデルにし、私たちが逆ベンチマークと呼ぶ技術を用います。ウィジェットがどのように実装されているかに関係なく、HashCore が衝突耐性を持つことの証明を提供します。GPP 設計者/開発者は本質的に SPEC CPU 2017 などのベンチマークのための ASIC を作成することを観察します。このようなベンチマークに基づいて HashCore をモデル化することで、GPP で最も効率的に実行できるプルーフ・オブ・ワーク関数を作成し、よりアクセスしやすく、競争的で、バランスの取れたマイニング市場を実現します。

[ieee.org](https://ieeexplore.ieee.org/document/8884814) で完全な論文をお読みください。
