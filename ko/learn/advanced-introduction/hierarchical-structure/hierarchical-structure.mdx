## 확장성 도전

전통적인 블록체인들은 불가능한 선택에 직면합니다: 작고 안전하게 유지할 것인가, 아니면 크지만 취약하게 성장할 것인가. Quai의 계층적 구조는 이러한 제한을 깨뜨립니다.

**혁신:**
모든 가지가 독립적으로 트랜잭션을 처리하지만 줄기가 모든 가지의 보안을 보장하는 나무를 상상해보세요. 그것이 Quai의 계층적 구조입니다 - 여러 블록체인 "가지들"이 전체 "나무"의 보안을 공유하면서 병렬로 작동합니다.

## 작동 방식

**세 단계의 조직:**
1. **프라임 체인** (줄기): 전체 네트워크를 조정
2. **리전 체인** (주요 가지): 지리적 영역을 조직  
3. **존 체인** (잎): 실제 트랜잭션을 처리

**핵심 돌파구:**
- **전통적 샤딩**: 보안을 분할 (위험함)
- **Quai의 접근법**: 작업량만 분할 (안전함)

각 존은 [병합 마이닝](/ko/learn/advanced-introduction/merged-mining/overview)을 통해 전체 네트워크의 완전한 보안을 유지하면서 독립적으로 트랜잭션을 처리합니다.

## 이것이 중요한 이유

**사용자를 위해:**
- **50,000+ TPS**: 비자보다 더 많은 트랜잭션 처리
- **페니 이하 수수료**: 대규모 확장에서도 지속 가능
- **5초 블록**: 거의 즉시 트랜잭션 확인
- **글로벌 규모**: 수십억 사용자를 위한 준비

**네트워크를 위해:**
- **무제한 성장**: 수요 증가에 따라 새로운 존 추가
- **유지된 보안**: 모든 존이 완전한 네트워크 보호를 받음
- **최적화된 성능**: 마이너들이 최고 효율을 위해 자체적으로 조직화

## 보안 혁신

**전통적 샤딩 문제:**
블록체인이 샤드로 분할될 때, 각 샤드는 일반적으로 보안의 일부분만 받습니다. 10개의 샤드가 있는 네트워크는 각 샤드에 총 보안의 10%만 제공할 수 있어 - 공격에 취약하게 만듭니다.

**Quai의 해결책:**
[병합 마이닝](/ko/learn/advanced-introduction/merged-mining/overview)을 통해, 모든 존의 모든 트랜잭션이 네트워크 보안의 100%를 받습니다. 이는 무제한 병렬 처리를 허용하면서 완전한 해시 기반 보안 보장을 유지합니다.

## 자체 최적화 성능

**지연 문제:**
[네트워크 지연](/ko/learn/advanced-introduction/hierarchical-structure/latency) - 노드 간 데이터 이동에 걸리는 시간 - 은 블록체인 성능의 주요 병목현상입니다.

**Quai의 독특한 해결책:**
마이너들이 자연스럽게 최적 성능을 위해 스스로를 조직화하는 이유:
1. **높은 지연 = 낮은 수익** (더 많은 엉클 블록)
2. **마이너들이 자신의 존을 선택** 최고 연결에 기반하여
3. **결과**: 중앙 계획 없는 지리적 자체 조직화

**개선의 수학적 근거:**

이 관계는 t = p*n*log_m(n)로 설명됩니다. 여기서:
- **t** = 전파까지의 시간
- **p** = 노드 간 핑 시간
- **n** = 네트워크의 노드 수
- **m** = 연결된 피어의 평균 수

**실제 예시:**
- **비트코인**: 8홉 × 100ms 핑 = 800ms 총 전파 시간
- **Quai 존들**: 3홉 × 10ms 핑 = 30ms 총 전파 시간
- **개선**: 25배 빠른 데이터 전파

이 자체 조직화 시스템은 중앙 조정 없이 자연스럽게 최적화된 지역 네트워크를 만듭니다.

## 병렬 처리 능력

Quai를 [블록체인을 위한 멀티코어 프로세서](/ko/learn/advanced-introduction/multithreaded-execution)로 생각해보세요. 각 존 체인은 독립적인 CPU 코어처럼 작동하며, 완벽하게 동기화된 상태로 트랜잭션을 병렬로 처리합니다.

**체인들의 협력 방식:**
- **독립적 운영**: 각 존이 자체 스케줄에 따라 블록 생성
- **완벽한 조정**: [해시 링크된 참조](/ko/learn/advanced-introduction/merged-mining/coincident-blocks)가 모든 체인을 동기화
- **원자적 연산**: 크로스체인 트랜잭션이 완전히 완료되거나 전혀 완료되지 않음

**3계층 아키텍처:**

| 체인 유형 | 계층 수준      | 네트워크 해시레이트 % | 블록 시간† |
| -------- | ------------- | ------------------- | --------- |
| 프라임    | 1 (최상위)     | 100%                | ~20초     |
| 리전     | 2 (중간)       | ~33%                | ~10초     |
| 존       | 3 (최하위)     | ~11%                | ~5초      |

† _Quai 네트워크의 블록 생성은 확률론적 과정으로, 실제 블록 시간은 약간의 변동이 있을 것입니다. 이 시간들은 2×2 네트워크 구성(2개 리전, 리전당 2개 존, 총 4개 존)에 대해 계산되었습니다._

### 간단하게 시작: 단일 샤드 런칭

**왜 하나로 시작하나?**
Quai 네트워크는 불필요한 샤드를 추가하는 것이 다음과 같은 문제를 일으키기 때문에 단일 존 체인으로 시작합니다:
- **집계 보안 감소**: 필요하지 않을 때 해시레이트 분할
- **정산 시간 증가**: 혜택 없는 복잡성
- **자원 낭비**: 사용되지 않는 용량을 위한 인프라 운영

**자동 확장 트리거:**
지속된 수요로 인해 엉클 비율이 연장된 기간 동안 20%를 초과하면, 네트워크는 새로운 샤드를 추가하여 자동으로 확장됩니다. 이는 성장의 모든 단계에서 최적의 보안과 성능을 보장합니다.

### 프라임 체인: 글로벌 조정자

**목적**: 네트워크의 백본 역할을 하여, 모든 리전이 동기화된 상태를 유지(여러 개가 존재할 때)
- **보안**: 최대 보안을 위해 네트워크 해시레이트의 100%로 마이닝
- **기능**: 순수한 조정 - 트랜잭션이나 계정 없음
- **역할**: 네트워크가 단일 샤드를 넘어 확장될 때 활성화

### 리전 체인: 지리적 조직

**목적**: 최적 성능을 위해 지리적 영역 내의 존들을 조직(확장 중 활성화)
- **보안**: 각 리전이 총 해시레이트의 ~1/N을 받음 (N = 리전 수)
- **기능**: 자신의 리전 내 존 체인들을 조정
- **활성화**: 네트워크 수요가 확장을 트리거할 때 생성

### 존 체인: 트랜잭션이 발생하는 곳

**목적**: 실제 작업자들 - 모든 사용자 트랜잭션을 처리
- **속도**: 거의 즉시 확인을 위한 5초 평균 블록 시간
- **기능**: 계정을 저장하고 트랜잭션을 처리하는 유일한 체인
- **초기 상태**: 네트워크가 하나의 존 체인으로 런칭
- **확장**: 수요에 기반하여 추가 존들이 추가됨

각 존은 독립적인 실행 스레드로 작동합니다. 네트워크가 성장함에 따라, 이는 진정한 [멀티스레드 블록체인 환경](/ko/learn/advanced-introduction/multithreaded-execution)을 만듭니다.

## 크로스체인 통신

**혁신:**
[일치 블록](/ko/learn/advanced-introduction/merged-mining/coincident-blocks)을 통해, Quai는 네트워크의 모든 체인 간에 자연스러운 브리지를 만듭니다. 이러한 해시 링크된 참조들은 추가 인프라 없이 무신뢰 크로스체인 트랜잭션을 가능하게 합니다.

**작동 방식:**
- **자동 생성**: 병합 마이닝이 자연스럽게 이러한 연결을 생성
- **제로 오버헤드**: 마이너들로부터 추가 작업이 필요 없음
- **무신뢰 전송**: 중개자 없이 체인 간 자산 이동
- **원자적 연산**: 크로스체인 트랜잭션이 완전히 완료되거나 전혀 완료되지 않음

이는 해시 기반 블록체인 간의 최초의 진정으로 탈중앙화된 브리지를 나타내며, 전체 네트워크에서 원활한 상호 운용성을 가능하게 합니다.
